# 🎨 LIFF 開發模式 - 測試連結

## 📱 主要頁面

### 1. 長者列表
```
http://localhost:5173/liff
```
- 顯示 4 位長者的卡片
- 管理員可以看到「新增長者」按鈕
- 可以點擊任何長者卡片進入詳情頁

### 2. 警報列表
```
http://localhost:5173/liff/alerts
```
- 顯示 4 筆警報記錄
- 可以篩選：全部/待處理/已確認/已解決
- 包含不同類型：緊急、電量不足、不活動、裝置離線

## 🎯 測試場景建議

### 場景 1：查看線上長者
1. 訪問長者列表
2. 點擊「王大明」或「張秀英」
3. 查看完整的個人資料和活動記錄

### 場景 2：查看警報狀態
1. 訪問警報列表
2. 查看不同嚴重程度的警報
3. 測試篩選功能（待處理/已確認/已解決）
4. 點擊「陳志強」的長者詳情，查看警報來源

### 場景 3：查看離線長者
1. 訪問長者列表
2. 點擊「李美華」（離線狀態）
3. 查看較少的活動記錄
4. 注意「最後上線」時間顯示

### 場景 4：管理員功能
1. 在長者列表頁面查看「新增長者」按鈕
2. 頂部導航欄顯示「管理員」標籤
3. 所有功能都可見

### 場景 5：響應式設計測試
1. 開啟瀏覽器開發者工具
2. 切換到手機模式（375px）
3. 測試所有頁面的手機版面
4. 測試平板模式（768px）

## 🔄 頁面導航流程

```
長者列表 (/liff)
  ├─→ 點擊長者卡片 → 長者詳情 (/liff/elder/:id)
  │                    └─→ 返回按鈕 → 回到長者列表
  │
  └─→ 點擊「警報記錄」標籤 → 警報列表 (/liff/alerts)
                              └─→ 點擊「長者列表」標籤 → 回到長者列表
```

---

## 📊 模擬資料統計

- **長者總數**：4 位
  - 線上：2 位（王大明、張秀英）
  - 離線：1 位（李美華）
  - 警報：1 位（陳志強）

- **警報總數**：4 筆
  - 待處理：2 筆（1 緊急、1 中度）
  - 已確認：1 筆
  - 已解決：1 筆

- **活動記錄**：自動生成
  - 每位長者都有 24 小時內的活動記錄
  - 記錄數量根據狀態不同

## 💡 開發提示

1. **修改樣式後**：瀏覽器會自動熱重載
2. **清除快取**：使用無痕模式或 Cmd+Shift+R
3. **查看 Console**：開啟開發者工具查看調試訊息
4. **模擬不同狀態**：可以修改 `LiffApp.tsx` 中的 `mockElders` 資料
